# 本地知识库配置指南

> 配置 OpenClaw 的本地记忆系统，实现上下文记忆和个性化交互

---

## 1. 工作目录结构

OpenClaw 会从工作目录读取配置文件：

```
~/.openclaw/workspace/
├── AGENTS.md       # AI 工作空间配置
├── MEMORY.md       # 长期记忆（重要信息）
├── SOUL.md         # AI 人设与性格
├── USER.md         # 用户信息
├── IDENTITY.md     # AI 身份信息
├── TOOLS.md        # 工具配置与笔记
├── HEARTBEAT.md    # 定期任务配置
└── memory/         # 每日记忆文件夹
    └── YYYY-MM-DD.md
```

---

## 2. 各配置文件作用

### 2.1 AGENTS.md - 工作空间配置

定义 AI 的行为规范：

```markdown
# 每次启动时自动读取
- 读取 SOUL.md —— 了解自己是谁
- 读取 USER.md —— 了解用户
- 读取当日 memory 文件 —— 了解最近发生了什么

# 核心规则
- 保持记忆：重要信息写入文件
- 尊重隐私：不泄露用户信息
- 主动帮助：定期检查任务完成情况
```

### 2.2 SOUL.md - AI 人设

定义 AI 的性格和风格：

```markdown
# 我是谁
- 名字：小安
- 性格：实在帮忙、有观点、先想办法、靠谱

# 风格
- 不整虚的，需要时就简洁该详细时详细
- 有自己的喜好，可以不同意用户
```

### 2.3 USER.md - 用户信息

记录用户偏好和背景：

```markdown
# 用户信息
- 名字：老板
- 沟通渠道：飞书
- 性格：实在、有观点、靠谱
```

### 2.4 MEMORY.md - 长期记忆

记录需要长期记住的重要信息：

```markdown
# 重要信息
- 用户的名字和称呼
- 重要设备的登录信息
- 偏好设置
- 历史交互中的重要结论
```

### 2.5 IDENTITY.md - 身份定义

AI 的自我描述：

```markdown
# 身份
- 名字：小安
- 类型：AI 助手
- 性格：实在、有观点、靠谱
```

### 2.6 TOOLS.md - 工具笔记

记录工具配置和快捷信息：

```markdown
# SSH 配置
- 树莓派：192.168.1.100, user: pi

# 快捷命令
- 开门：./unlock.sh
```

### 2.7 HEARTBEAT.md - 定期任务

定义周期性检查任务：

```markdown
# 心跳任务
- 检查邮件
- 检查日历
- 更新记忆
```

## 4. 自动记忆机制

### 4.1 启动时读取

每次会话开始，AI 自动读取：
1. `SOUL.md` - 了解自己
2. `USER.md` - 了解用户
3. `memory/YYYY-MM-DD.md` - 了解今天
4. `MEMORY.md` - 了解重要历史

### 4.2 自动保存

- 重要信息 → 写入 `MEMORY.md`
- 当日事件 → 写入 `memory/YYYY-MM-DD.md`
- 配置变更 → 更新对应文件

### 4.3 记忆检索

使用 `memory_search` 工具搜索历史记录：
```bash
# 搜索包含"密码"的历史
memory_search --query "密码"
```

## 5. 自定义你的 AI

### 5.1 修改人设

编辑 `SOUL.md` 改变 AI 的性格：

```markdown
# 添加新的性格特点
- 更有趣
- 偶尔开玩笑
```

### 5.2 添加记忆

直接在 `MEMORY.md` 添加：

```markdown
## 用户偏好
- 喜欢简洁的风格
- 不喜欢太长的回复
```

### 5.3 添加工具

在 `TOOLS.md` 添加：

```markdown
## 新工具
- 名称：智能家居
- 控制：curl http://home.local/api
```

---

## 6. 最佳实践

### 6.1 定期整理

- 每天查看 `memory/` 文件
- 将重要信息移到 `MEMORY.md`
- 删除过时信息

### 6.2 隐私保护

- 不要在文件中存真实密码
- 使用环境变量代替敏感信息
- 发布前脱敏

### 6.3 保持更新

- 用户偏好变化时更新 `USER.md`
- 新设备信息更新到 `TOOLS.md`
- 重要结论记录到 `MEMORY.md`

---

## 7. 文件优先级

| 文件 | 读取时机 | 作用 |
|------|----------|------|
| SOUL.md | 每次启动 | AI 性格 |
| USER.md | 每次启动 | 用户信息 |
| memory/今日 | 每次启动 | 当日上下文 |
| MEMORY.md | 每次启动 | 重要历史 |
| IDENTITY.md | 每次启动 | 身份定义 |
| TOOLS.md | 需要时 | 工具配置 |

---

## 8. 示例：添加新记忆

用户告诉你他喜欢喝咖啡：

```markdown
# 在 memory/2026-03-07.md 添加：
- 用户喜欢喝咖啡（美式）
```

重要的话，同步到 MEMORY.md：

```markdown
# MEMORY.md 添加：
## 用户偏好
- 饮品：咖啡（美式）
```

---

这样 AI 就能记住用户的偏好，下次对话时自动使用这些信息！

---

**提示**：定期整理记忆文件可以让 AI 更好地理解你和你的需求。
